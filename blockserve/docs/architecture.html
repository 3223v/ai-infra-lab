📥 WorkloadLoader

  • 逐行解析 JSONL 文件
  • 验证必填字段：arrival_time、prompt_len、max_new_tokens
  • 自动分配 Request ID（从 1 递增）
  • 跳过空行和注释行
  • 返回 WorkloadLoadResult（requests + errors）
  • 调用方必须先检查 ok()

📦 Request

  • 7 种状态：WAITING → PREFILLING → DECODING → FINISHED / CANCELED / REJECTED / TIMED_OUT
  • is_terminal_status() 判断终态
  • to_string() 状态转字符串
  • 当前模拟阶段仅使用 WAITING 状态

⏳ Simulator

  • step() — 推进 1 个时间单位
  • run_until(max_time) — 自动推进到上限或全部到达
  • 构造时按 arrival_time 升序排列请求
  • 每 tick 将到达请求从 pending → waiting_queue
  • all_arrived() 判断是否全部到达

🗂 RequestQueue

  • 基于 std::deque<Request> 的薄封装
  • pop() 返回 std::optional（空队列安全）
  • front() 返回 const 指针，不弹出
  • 后续将被 Scheduler 消费

🔜 待实现（Day 5+）

  • Scheduler — 从 waiting_queue 取请求，执行状态迁移
  • BlockPool — KV Cache block 分配/回收
  • RequestBlockTable — 每请求 block 映射
  • MetricsRecorder — TTFT / TPOT / throughput

📐 调用关系

  • main → Simulator(requests)
  • Simulator::step → RequestQueue::push
  • Simulator::step → log::print_request
  • Simulator 持有 pending_requests_（vector）和 waiting_queue_（RequestQueue）
